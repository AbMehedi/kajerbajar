# Supabase Migrations Guide

This directory tracks incremental database schema changes per phase.

## How to Use

1. **Run once (on first setup):**
   ```sql
   -- Supabase SQL Editor → Copy + Paste → Run
   supabase/schema.sql
   ```

2. **Run migrations in order (one at a time):**
   ```sql
   supabase/migrations/001_phase1_auth.sql         # Phase 1.1
   supabase/migrations/002_company_verification.sql # Phase 1.2
   supabase/migrations/003_skill_verification.sql   # Phase 2.1
   -- etc...
   ```

## Manual: How to Create Migration SQL

When adding new features:

```sql
-- supabase/migrations/00X_[feature].sql

-- 1. Add new columns
ALTER TABLE table_name ADD COLUMN new_column TYPE;

-- 2. Create indexes if needed
CREATE INDEX idx_table_new_column ON table_name(new_column);

-- 3. Add RLS policies
CREATE POLICY "[name]" ON table_name 
  FOR [INSERT|UPDATE|DELETE|SELECT] 
  TO authenticated 
  USING ([condition]) 
  WITH CHECK ([condition]);

-- 4. Add helper functions if needed
CREATE OR REPLACE FUNCTION function_name()
RETURNS type AS $$ ... $$ LANGUAGE sql;
```

If cloning the repo:

1. Run `supabase/schema.sql` once
2. Run migrations 002, 003, etc. in order
3. Each migration is self-contained and idempotent (safe to re-run)
